# CLAUDE.md

`autofish_freetime` 前端，Next.js + React + Tailwind CSS v3。

## Tab 页面设计规范

参考实现：`app/dashboard/settings/page.tsx`

### 通用规则

**1. 内外分层**

一级 tab 栏放在卡片容器**外部**，tab 内容放在卡片**内部**。利用容器的物理边界（`border` + `shadow-sm`）天然建立层级关系——用户不需要思考就能感知「外面的是主导航，里面的东西归它管」。

一级 tab 栏与卡片之间用 `space-y-5`（20px）分隔，足够区分但不断裂。

**2. 重上轻下**

视觉重量从上到下递减。上层元素字体更大更粗、颜色更饱和；下层元素更小更轻。具体比例：

- 一级 tab：`text-base font-semibold`，激活时用品牌色（`text-blue-600`）
- 筛选控件/二级切换：`text-sm font-medium`，激活时用浅底（`bg-blue-50 text-blue-700`），不抢一级 tab 的注意力
- 数据区：`text-xs` ~ `text-sm`，灰色为主，最低视觉重量

原则：站在页面远处眯眼看，一级 tab 应该是第一个跳入眼睛的东西。

**3. 语义区分色重**

不同语义的交互元素用不同的色彩重量，避免撞脸：

- **导航/筛选**（选了哪个）：浅底深字，如 `bg-blue-50 text-blue-700`——轻量指示，不呼出
- **操作按钮**（要做什么）：实心底色白字，如 `bg-blue-600 text-white`——全重色，呼出行动
- **危险操作**：延续红色体系，确认态实心、普通态浅字

**4. 细分不抢眼**

卡片内部的分割线用 `border-gray-100`（不是 200、300），够用即可。分割线的目的是引导视线，不是框死内容。越轻的线，内容的一体感越强。

**5. 少即是多**

一级 tab 的标签本身充当内容区标题。不要再额外加 h2 标题重复 tab 名称——用户看到同一个名字出现两次会产生困惑。如果 tab 名称已经说明了内容，就让它做唯一的标题。

### 反模式

- ❌ 二级控件比一级 tab 更鲜艳/更大/更粗
- ❌ 筛选 pill 和操作按钮用同一套蓝底白字样式
- ❌ tab 内再加一个重复 tab 名称的标题
- ❌ 分割线用 `border-gray-200` 或更重——内部线条应该轻于外部边框
- ❌ tab 栏和内容区贴在一起，没有容器边界
