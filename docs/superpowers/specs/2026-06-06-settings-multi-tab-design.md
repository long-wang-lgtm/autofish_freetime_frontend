# Settings 页面多 Tab 改造设计

**日期**: 2026-06-06  
**范围**: `frontend/app/dashboard/settings/page.tsx`

## 目标

将设置页面从单一 AI 配置页改为支持多 tab 切换的页面结构，为后续添加其他设置模块做好准备。

## 设计方案

采用纯 state 驱动的 tab 切换（方案 A），无需改动路由结构。

### 页面结构

```
┌──────────────────────────────────────────────┐
│ 设置                                         │
│ ┌──────────┬──────────┬──────────┐           │
│ │ 🤖 AI配置 │ 📦 Tab 2  │ 📦 Tab 3  │  ← 左上角  │
│ └──────────┴──────────┴──────────┘           │
│  ─────────────────────────────────  (分隔线)  │
│                                              │
│  [当前 AI 配置页面的全部内容]                    │
│  - 标题 + 添加按钮                             │
│  - all/text/image 子过滤                      │
│  - 配置列表表格                                │
└──────────────────────────────────────────────┘
```

### Tab 定义

| Tab Key | 标签 | 内容 |
|---------|------|------|
| `ai-config` | 🤖 AI 配置 | 当前 AI 配置列表全部内容 |
| `tab-2` | 待定 | 占位空状态 |
| `tab-3` | 待定 | 占位空状态 |

### 改动点

1. **页面标题**: 「AI 配置」→「设置」
2. **一级 Tab 栏**: 页面顶部左侧渲染 tab 按钮组，使用 `useState<'ai-config' | 'tab-2' | 'tab-3'>` 管理切换
3. **内容区域**: 根据 `activeTab` 条件渲染不同面板
4. **AI 配置 tab**: 将现有全部内容（标题、添加按钮、子过滤、表格）完整保留
5. **占位 tab**: 显示空状态提示「即将上线」
6. **子路由不受影响**: `add/` 和 `[id]/` 页面独立，不需要改动

### 样式约定

- Tab 按钮使用底部高亮条样式（类似浏览器 tab），与当前项目风格一致
- 选中 tab：蓝色文字 + 蓝色底部边框
- 未选中 tab：灰色文字 + hover 效果
- Tab 栏无背景色，融入页面

### 不影响的范围

- `app/dashboard/settings/add/page.tsx` — 添加页面不变
- `app/dashboard/settings/[id]/page.tsx` — 编辑页面不变
- `app/dashboard/layout.tsx` — 布局不变
- `lib/api/ai-config.ts` — API 层不变
