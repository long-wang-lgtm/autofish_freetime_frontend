# 色彩语义体系

> 参考实现：`styles/globals.css`（CSS 变量定义）、`tailwind.config.js`（语义 Token 映射）

## 核心原则

### 1. 品牌色唯一

品牌色（`blue-600` / `--primary`）**仅**用于 CTA 操作按钮和品牌标识（Logo、侧边栏高亮），不用于数据指标、普通文本链接、信息徽章或背景装饰。

- CTA 按钮是唯一可以使用实心品牌色的场景
- 数据指标使用中性色（`text-gray-900`），不使用品牌色

### 2. 交互状态三色体系

导航/筛选的交互状态使用三套独立的颜色，绝不混用：

| 状态 | 配色 | 用途 |
|------|------|------|
| **选中/激活** | `bg-blue-50 text-blue-700` | Tab 激活、筛选项选中、导航当前页 |
| **悬停** | `bg-gray-50` | 表格行 hover、卡片 hover、下拉选项 hover |
| **聚焦** | `ring-2 ring-blue-500` | 输入框 focus、按钮 focus-visible |

筛选 pill 用浅底深字（bg-blue-50 text-blue-700），不用实心底色白字。筛选 pill 和操作按钮应使用不同的颜色样式，避免混淆。

### 3. 语义状态色

操作结果、数据状态、系统反馈使用独立于品牌色的语义色系：

| 语义 | 配色 | 典型场景 |
|------|------|----------|
| **成功 / 正常 / 启用** | `green-600`（文字）/ `emerald-500`（图标） | 在线状态、操作成功提示、启用标记 |
| **警告 / 待处理** | `amber-500`（浅底）/ `amber-600`（文字） | 待审核、即将过期、需要关注 |
| **错误 / 禁用 / 异常** | `red-500`（浅底）/ `red-600`（文字） | 删除按钮、错误提示、离线状态 |
| **信息 / 中性** | `gray-400`（图标）/ `gray-500`（文字） | 次要信息、帮助提示、无数据占位 |

品牌色（蓝色）不代表"正常"或"警告"等状态语义，状态色必须按语义选择对应的独立色系。

### 4. 灰度色阶分层

每个灰度色阶有且仅有一种核心用途，严格从上到下递减使用：

| 色阶 | 用途 | 说明 |
|------|------|------|
| `gray-50` | **页面背景**（唯一） | 仅用于最外层页面底色 |
| `gray-100` | **表头背景 / 禁用态背景 / 分割线**（唯一） | 卡片内部分割线使用 `border-gray-100` |
| `gray-200` | **卡片边框 / 输入框边框**（唯一） | 组件的物理边界 |
| `gray-300` | 占位符文字 | placeholder 颜色 |
| `gray-400` | 辅助图标、次要装饰 | 最低对比度级 |
| `gray-500` | **次要正文** | 描述文字、标签、注释 |
| `gray-600` | **辅助标题** | 列表副标题、卡片副标题 |
| `gray-700` | **正文** | 主要阅读文字 |
| `gray-800` | **加粗正文 / 重要标题** | 卡片标题、表单标签 |
| `gray-900` | **主标题**（唯一） | 页面标题、Section 标题 |

输入框背景应使用白色（与页面背景区分），不可使用 `gray-50`。分割线使用 `border-gray-100`，不可使用 `border-gray-200` 或更深的灰色。

### 5. 图表色（独立体系）

图表数据系列的颜色与 UI 交互色完全独立，不共用品牌蓝。防止用户将图表中的蓝色数据系列误认为可点击的 CTA 按钮。

统一的图表指标色映射：

| 指标 | 颜色 | 色值 | 说明 |
|------|------|------|------|
| 想要 | 蓝 | `#2563eb` | 核心转化指标 |
| 浏览 | 琥珀 | `#d97706` | 流量指标 |
| 收藏 | 紫罗兰 | `#7c3aed` | 兴趣指标 |
| 询单率 | 翠绿 | `#059669` | 转化率指标 |

图表数据系列色与 UI 按钮色必须分离，不可混用。

### 6. toaster 颜色

toaster 通知的 variant 颜色对照表（基于 Tailwind 默认色板）：

| variant | 背景 | 边框 | 文字 |
|---------|------|------|------|
| error | `#fee2e7`（red-100） | `#f43f5e`（rose-500） | `#be123c`（rose-700） |
| success | `#dcfce7`（green-100） | `#22c55e`（green-500） | `#166534`（green-700） |
| warning | `#fef3c7`（amber-100） | `#f59e0b`（amber-500） | `#92400e`（amber-800） |
| info | `#dbeafe`（blue-100） | `#3b82f6`（blue-500） | `#1e40af`（blue-800） |

### 7. 主题切换

项目必须支持用户手动切换日间/夜间主题，默认跟随系统偏好。

#### 切换机制

- **存储**：用户选择持久化到 localStorage，key 为 `theme`，值为 `light` | `dark` | `system`
- **默认值**：未选择时默认 `system`，跟随 `prefers-color-scheme` 媒体查询
- **切换入口**：Header 右侧用户菜单中放置主题切换按钮（日间/夜间图标 toggle）
- **即时生效**：切换时在 `<html>` 元素上添加/移除 `dark` class，Tailwind 的 `darkMode: "class"` 策略驱动所有 `dark:` 前缀样式

#### 日间 / 夜间色板对照

> 夜间主题遵循"降低亮度、保留色相、减小对比度跨度"原则。以下为色彩变量的两套映射。

**背景与表面：**

| 层级 | 日间 | 夜间 |
|------|------|------|
| 页面底色 | `bg-gray-50` | `bg-gray-950` |
| 卡片 / 面板 | `bg-white` | `bg-gray-900` |
| 表头 / 禁用态 | `bg-gray-100` | `bg-gray-800` |
| 卡片边框 | `border-gray-200` | `border-gray-700` |
| 分割线 | `border-gray-100` | `border-gray-800` |

**文字：**

| 层级 | 日间 | 夜间 |
|------|------|------|
| 主标题 | `text-gray-900` | `text-gray-100` |
| 正文 | `text-gray-700` | `text-gray-300` |
| 辅助文字 | `text-gray-500` | `text-gray-400` |
| 禁用/占位 | `text-gray-400` | `text-gray-500` |

**交互状态（夜间维持色相、降低饱和度）：**

| 状态 | 日间 | 夜间 |
|------|------|------|
| CTA 按钮 | `bg-blue-600 text-white` | `bg-blue-500 text-white` |
| 选中/激活 | `bg-blue-50 text-blue-700` | `bg-blue-950 text-blue-300` |
| 悬停 | `bg-gray-50` | `bg-gray-800` |
| 聚焦 ring | `ring-blue-500` | `ring-blue-400` |

**语义色（夜间小幅提亮以维持可辨度）：**

| 语义 | 日间 | 夜间 |
|------|------|------|
| 成功 | `text-green-600` / `bg-green-100` | `text-green-400` / `bg-green-950` |
| 警告 | `text-amber-600` / `bg-amber-50` | `text-amber-400` / `bg-amber-950` |
| 错误 | `text-red-600` / `bg-red-100` | `text-red-400` / `bg-red-950` |

#### 实施策略

1. **基础设施**（Phase 1）：
   - 创建 `hooks/useTheme.ts` — 读取 localStorage / 系统偏好 / 切换 `<html>` class
   - Header 添加主题切换按钮
   - 修复 `tailwind.config.js` 确认 `darkMode: "class"`
2. **逐页适配**（后续 Phase）：
   - 每个页面的组件添加 `dark:` 前缀
   - 优先适配高频页面：商品管理、账号管理、选品监控
   - 图表组件在夜间模式下切换 ECharts 暗色主题（`echarts.init(dom, 'dark')`）

#### 铁律

- **禁止对日间色值做任何假设**：所有颜色必须同时提供日间和夜间值（通过 `dark:` 前缀）。
- **夜间色值不简单地反转**：不是 `bg-white → bg-black`，而是 `bg-white → bg-gray-900`，保留微妙的层次感。
- **语义色保持色相**：绿色在夜间仍是绿色（不是简单地变亮变暗），仅调整明度/饱和度。
- **新组件必须同时定义日间和夜间样式**：从第一个组件开始就写 `dark:` 前缀，不允许事后补。

## 反模式

- 用品牌色表示数据指标数值（应使用中性色）
- 筛选 pill 使用与 CTA 按钮相同的实心底色白字样式（应使用浅底深字）
- 用品牌色表示"在线/成功/正常"等状态（应使用绿色系）
- 用品牌色表示"警告/待处理"（应使用琥珀色系）
- `gray-50` 同时用于页面背景和输入框背景（应严格区分）
- 分割线使用 `border-gray-200` 或更重（应使用 `border-gray-100`）
- 图表数据系列色与 UI 按钮色混用（图表应使用独立色值体系）
- 新组件绕过 `bg-primary` 直接用 `bg-blue-600`（应优先使用语义 Token）
- `.dark` CSS 变量留在代码中但永不启用（要么删除要么标注待实现）
